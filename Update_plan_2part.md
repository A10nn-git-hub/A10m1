 План разработки и исправления ошибок: 2D Shooter Telegram Bot

Этот документ содержит перечень технических задач для оптимизации и исправления багов в игре. Каждая тема является изолированным спринтом. Перед выполнением задач внутри темы необходимо инициализировать специализированного контекстного агента.

## Тема 1: Инициализация, Загрузочный экран и Пасхалки
> **Системная команда:** `[Действие: Создай нового изолированного агента для оптимизации логики сплэш-скрина и интеграции скрытых функций]`

* **Задача 1.1: Корректировка таймингов сплэш-скрина (заглушки)**
  * **Текущее поведение:** Индикатор загрузки некорректно стартует сразу с 13%.
  * **Требуемое поведение:** При запуске приложения прогресс-бар должен строго удерживать значение **0% в течение 2 секунд** (для завершения фоновых процессов инициализации), и только после этого плавно продолжать загрузку.

* **Задача 1.2: Интеграция пасхалки для быстрого старта (Режим разработчика)**
  * **Алгоритм активации:** Двойной тап по элементу интерфейса `Мишень` -> последующий непрерывный свайп слева направо по текстовой надписи `2d shooter`.
  * **Логика работы:** Данный триггер должен мгновенно и полностью пропускать загрузочную заглушку (Fast Start). Режим носит исключительно утилитарный характер: он **не влияет** на игровой баланс и **не открывает** дополнительные панели разработчика. Единственная функция — экономия времени при тестах.

## Тема 3: Игровая камера и Системы управления (Inputs)
> **Системная команда:** `[Действие: Создай нового изолированного агента для калибровки камеры, ортографической проекции и дебага тач-событий UI]`

* **Задача 3.1: Масштабирование тактического экрана**
  * **Описание:** Исправить логику отображения на экране «ТАКТИЧЕСКАЯ КАРТА». Карта уровня должна автоматически масштабироваться (Fit/Contain) так, чтобы в границах модального окна была видна **вся игровая зона целиком**, а не ее обрезанный фрагмент.


* **Задача 3.2: Восстановление работоспособности контроллеров персонажа**
  * **Проблема:** Полная потеря контроля над бойцом. Графические элементы управления (джойстик движения, зоны взаимодействия) визуально отсутствуют на экране. При слепом нажатии на дефолтные области размещения инпутов события не регистрируются. Сброс раскладки в меню кастомизации интерфейса не решает проблему.
  * **Вектор проверки:** Проверить слои рендеринга интерфейса (Z-index/Canvas Sort Order), инициализацию Touch Events и логику подгрузки пресетов управления при старте игровой сцены.

## Тема 4: Игровая сессия, Стейт-машина раундов и Звуковой движок
> **Системная команда:** `[Действие: Создай нового изолированного агента для дебага жизненного цикла игровых сессий и логики аудио/визуальных эффектов]`

* **Задача 4.1: Устранение бага каскадного сброса раундов**
  * **Проблема:** После фиксации первого проигранного раунда стейт-машина ломается, из-за чего все последующие раунды автоматически завершаются/сбрасываются в начальное состояние, не позволяя игроку продолжить полноценную сессию. Необходимо проверить методы валидации условий победы/поражения и очистки стейта между раундами.
  * **Видеофиксация бага:** `видео 1.2` (файл `230.mp4`) — баг с автоматическим сбросом последующих раундов.

* **Задача 4.2: Исправление звукового сопровождения и анимации финала раунда**
  * **Аудио-система:** Заменить некорректный низкочастотный гул/помеху при окончании раунда на воспроизведение штатных аудиофайлов победы команд ("Terrorists Win" / "Counter-Terrorists Win"). Проверить корректность путей к аудио-записям в ассетах.
  * **Визуальная часть:** Полностью переработать плашку уведомления о победе. Вместо мелкого статичного текста в верхней части экрана реализовать UI-элемент по центру. Текст должен быть крупным и использовать логику анимации предматчевого отсчета "3..2..1": появление в центре экрана -> динамическое уменьшение масштаба (scale down) -> плавное исчезновение (fade out).

### 🛠 ТЕМА 1: Синхронизация обновлений и автоматическое версионирование через GitHub
> 🤖 **Действие:** _Создай нового агента для интеграции с GitHub API и управления конфигурацией лобби._

* **Логика работы:**
  * При каждом входе игрока в главное лобби должен срабатывать хук авторизации/инициализации, запрашивающий актуальный текст последнего коммита или файла обновлений напрямую из репозитория *GitHub*.
  * Если полученный текст изменений отличается от текущего кэшированного текста в базе данных, система должна автоматически инкрементировать версию игры на одну подверсию (на `0.01` для минорных или на `0.1` для промежуточных изменений).
* **Интерфейс настроек (Settings UX):**
  * В меню настроек необходимо реализовать динамический текстовый блок, куда автоматически транслируется текст коммита (лог обновления).
  * Рядом должна отображаться текущая автообновляемая строка версии (например, `v1.24`).

### 👥 ТЕМА 2: Интеллектуальная сортировка списка друзей
> 🤖 **Действие:** _Создай нового агента для переработки системы социальных связей и кастомных алгоритмов сортировки списка друзей._

* **Иерархия отображения:**
  * Необходимо полностью переработать компаратор списка друзей. При выводе списка пользователей, сортировка должна строго следовать следующему приоритету состояний (от высшего к низшему):
    1. 🤖 **ИИ (Боты)** — всегда отображаются в самом верху списка для быстрого тестирования и взаимодействия.
    2. 🟢 **В сети (Online)** — активные пользователи.
    3. 🟡 **Отошел (AFK)** — пользователи, сменившие статус активности.
    4. 🔴 **Не в сети (Offline)** — пользователи, которые отсутствуют, но заходили недавно.
    5. ⏳ **Не в сети [X] дней назад** — пользователи с сортировкой по возрастанию времени их последнего визита.

### 🐛 ТЕМА 4: Исправление критических багов и логики игрового процесса (Режимы 1х1, 5х5)
> 🤖 **Действие:** _Создай нового агента контроля качества (QA) и игровой логики для устранения багов в матчах._

* **Критические исправления (Bug Fixes):**
  * 🛑 **Разрыв соединения (Disconnect mid-game):** Если игрок полностью выключает девайс или теряет сеть прямо во время катки, таймер текущего раунда/отсчета замораживается. Прямо под табло со счетом должен выводиться стилизованный текст: `«Ожидание переподключения соперников...»` *(визуальный стиль аналогичен Standoff 2)*. При этом глобальный статус игрока в профиле меняется с *«Играет в...»* на *«Отошел»*.
  * 🔄 **Смена раскладки управления:** Исправить баг, при котором изменение кастомной раскладки кнопок во время матча намертво вешает игру и рендерит черный круг вместо карты. Игра должна бесшовно продолжаться в бэкграунде.
  * ⚔️ **Жизненный цикл раундов 1х1 (Mobile):** Устранить баг, при котором после намеренного проигрыша первого раунда, второй раунд автоматически пропускается, а у проигравшего отображается полноэкранная тактическая карта. *Полноэкранная карта должна показываться только тогда, когда у игрока остались живые тиммейты*, за которыми можно наблюдать в режиме спектатора. В 1х1 игрок обязан полноценно переходить к игре во 2-м раунде.
  * ❤️ **Синхронизация ХП у ботов:** Проверить и исправить все методы распределения здоровья (`HP`) у ИИ. Сейчас наблюдается десинхронизация: в разных раундах у ботов генерируется случайное количество здоровья (они становятся либо ваншотными, либо неуязвимыми боссами). Установить строгое базовое значение `100 HP`.
  * ⏳ **Оверлей возрождения в КБ (5х5):** В режиме «Командный бой» после смерти игрока, пока он ожидает респавна и видит общую карту, на экране должен отображаться графический таймер обратного отсчета до его возрождения.
  * 🛡 **Баланс спавн-щита:** Защитный щит при возрождении в КБ должен действовать ровно `5 секунд`, **ЛИБО** автоматически отключаться *в ту же миллисекунду, когда игрок совершает первое физическое движение* (инпут с джойстика/клавиатуры).

### 🎭 ТЕМА 5: Интерфейс выбора стороны и логика балансировки команд
> 🤖 **Действие:** _Создай нового агента для реализации UI/UX экрана выбора команд и алгоритмов распределения слотов._

* **Дизайн экрана (В стиле Standoff 2):**
  * При входе в любой режим (`1x1`, `2x2`, `5x5`) появляется полноэкранное модальное окно, разделенное вертикально ровно пополам:
    * **Левая половина (Синяя зона):** Оборона (CT).
    * **Правая половина (Оранжевая зона):** Атака (T).
  * Названия команд расположены сверху над зонами и отделены от них четкой горизонтальной линией. Внутри каждой зоны списком отображаются никнеймы реальных игроков, выбравших эту сторону. Выбор команды, базы и цвета стороны происходит в один клик по зоне.
* **Исключение ИИ из UI:**
  * В этой таблице выбора **НЕ должно быть ИИ (ботов)**. Они распределяются и заходят на оставшиеся пустые слоты автоматически *только после того, как все реальные игроки зафиксируют свой выбор*.
* **Логика строгого распределения слотов:**
  * Игроки могут заходить за команды строго по очереди для исключения жесткого дисбаланса сил (например: 1 в CT -> 1 в T; нельзя зайти за CT, если там уже 2 игрока, а в T — 0). Заполнение идет равномерно согласно лимитам режима: для 1х1 (макс 2 игрока), 2х2 (макс 4 игрока), 5х5 (макс 10 игроков).

### 🔊 ТЕМА 6: Обновление аудио-движка и замена звуковых ассетов
> 🤖 **Действие:** _Создай нового агента для управления звуковой системой, ресурсами и ивент-триггерами._

* **Удаление старой системы:**
  * Полностью вырезать и депрецировать прошлую голосовую озвучку побед, сгенерированную искусственным интеллектом. Использование ИИ-голоса для системных уведомлений признано архитектурной ошибкой.
* **Интеграция новых аудио-файлов:**
  * Подключить к звуковому движку два новых статических звуковых файла, переданных разработчиком:
    * Событие `COUNTER_TERRORISTS_WIN` -> Срабатывает воспроизведение **«Звук 1»**.
    * Событие `TERRORISTS_WIN` -> Срабатывает воспроизведение **«Звук 2»**.
