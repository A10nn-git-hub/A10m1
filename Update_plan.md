# 🛠️ План доработок и оптимизации: GEMINI ANTIGRAVITY
## Техническое задание / Обновление файла `README.md`

Данный документ содержит перечень необходимых исправлений, оптимизаций интерфейса (UI/UX) и расширения функционала для 2D-шутера в Telegram-боте.

---

### 1. 📋 Оптимизация игрового статуса (Rich Presence)
* **Текущая проблема:** Статус игрока в режиме командного боя выглядит перегруженным («играет в командный бой 5х5»).
* **Задача:** Изменить текстовый шаблон отображения активности.
* **Технические требования:**
    * Для режима **Командный бой** сократить строку до: `«Играет в командный бой»` (полностью убрать приписку формата матча *5х5*).
    * Для остальных игровых режимов (например, **Дуэли**) оставить текущую логику с выводом формата или количества участников (например, *1х1*, *2х2*), чтобы сохранять информативность.

---

### 2. 🎛️ Система шаблонов (пресетов) для кастомных настроек лобби
* **Суть изменения:** Внедрение быстрых шаблонов конфигурации матча, стандартизированных для всех пользователей. Игроки не могут редактировать сами шаблоны, но могут применять их в один клик для экономии времени.
* **Логика интерфейса:** При создании лобби и переходе в раздел *«Кастомные настройки»*, вместо классического детального меню, пользователю сначала открывается промежуточный выбор. Меню представляет собой три вертикальные кнопки (в столбик):
    1.  **`ПОБЕДНЫЙ`** *(Режим №1)*
    2.  **`СКОРОСТНОЙ`** *(Режим №2)*
    3.  **`ПОЛЬЗОВАТЕЛЬСКИЕ`** *(Режим №3)*

* **Спецификация режимов:**
    * **Режим №1 («ПОБЕДНЫЙ»):** У всех живых игроков (настоящих пользователей) все параметры выставляются на максимум (здоровье, урон, боезапас и т.д.). У ботов остаются оригинальные (базовые) настройки лобби. 
        * *Важно:* Описание характеристик скрыто, игроки видят только название пресета. При клике параметры меняются на максимальные прямо на глазах у создателя лобби.
    * **Режим №2 («СКОРОСТНОЙ»):** Устанавливает максимальную скорость передвижения и действий абсолютно для всех участников матча, включая ботов. Все остальные параметры остаются стандартными.
    * **Режим №3 («ПОЛЬЗОВАТЕЛЬСКИЕ»):** Перенаправляет пользователя на классическое детальное меню конфигурации, где можно вручную настроить каждый параметр для каждого игрока или бота. До нажатия этой кнопки старое громоздкое меню должно быть полностью скрыто.

---

### 3. ⚙️ Интеграция настройки чувствительности управления
* **Задача:** Расширить меню конфигурации игрового процесса индивидуальной настройкой управления.
* **Технические требования:**
    * Разместить новый параметр **«Чувствительность»** в разделе общих настроек, позиционировав его непосредственно *перед строкой с версией программы*.
    * **Диапазон значений:** от `0` до `10`.
    * **Стандартное значение:** По умолчанию (изначально) выставлять `5`, что эквивалентно текущей базовой скорости движения/прицеливания, с которой игроки ездят сейчас. Изменение значения должно плавно влиять на физику управления в матче.

---

### 4. 🎁 Управление архивными (удалёнными) предметами и промокодами
* **Суть изменения:** Предметы, выведенные из общего ассортимента игрового магазина, не должны вырезаться из кода. Они переходят в статус «архивных».
* **Технические требования:**
    * Исключить данные предметы из отображения во внутриигровом магазине (их нельзя купить за обычную валюту).
    * Сохранить полную поддержку этих предметов в самом игровом процессе (если они уже есть у игрока, они должны корректно работать).
    * В админ-панели разработчика реализовать функционал прямой выдачи данных предметов конкретным пользователям.
    * Добавить возможность привязки архивных предметов к пулу наград в системе генерации промокодов (`промиков`).

---

### 5. 🦎 Консервация контента кейса «Chameleon Box»
* **Суть изменения:** Перевод кейса в категорию редкого/коллекционного контента.
* **Технические требования:**
    * Удалить **Chameleon Box** из витрины внутриигрового магазина (прямая покупка становится невозможной).
    * Полностью сохранить внутреннее содержимое кейса: все оригинальные предметы, скины, имена и наградные медали должны остаться в лут-пуле без изменений.
    * Сохранить техническую возможность открытия кейса. Если у игрока есть этот бокс в инвентаре (или он получен через промокод/админку), механика открытия должна срабатывать штатно.

---

### 6. 🚨 Редизайн системы уведомлений (UI/UX) и скрытие рутинных окон
* **Текущая проблема:** Наличие лишних всплывающих окон Telegram или системных алертов на каждое успешное действие (например, текст «ОТПРАВЛЕНО» при инвайте в лобби), что замедляет игровой процесс и раздражает пользователя.
* **Задача:** Полный перенос рутинных уведомлений в фоновый режим и внедрение кастомного компактного UI для критических событий.
* **Технические требования:**
    * Полностью убрать всплывающие окна при успешных действиях (приглашение отправлено, лобби создано и т.д.). Никаких уведомлений в таких сценариях выводить не нужно.
    * Разработать и внедрить **фирменный компактный виджет уведомлений**.
    * **Visual-style виджета:** Маленькое, аккуратное окно с красивым переливающимся неоновым/светящимся градиентом зеленого цвета (*green glow/shimmer effect*).
    * **Логика работы:** Виджет активируется **только при возникновении ошибок или предупреждений**. Внутри отображается лаконичный текст проблемы.
    * **Примеры системных ошибок для нового окна:**
        * `«Игра заполнена»` — при попытке входа в лобби, где уже достигнут лимит игроков.
        * `«Некорректный ID»` — при вводе несуществующего или ошибочного ID при поиске матча/игрока.
